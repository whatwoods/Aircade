# Aircade 产品需求文档（PRD）

**版本**：v1.0（最终版）
**项目名**：Aircade
**定位**：《AI 做游戏》群友作品展示站——群友们亲手造的街机厅
**文档作者**：Way
**最后更新**：2026-04-17

---

## 1. 背景与目标

群内群友持续产出微信小程序/小游戏和部分 web 应用，目前散落在聊天记录里，看不到、留不住、传不开。Aircade 是一个集中展示站，用来沉淀作品、促进群友互动、顺带带来外部流量和新群友。

**北极星指标**：
- 月度活跃访客中群友占比 ≥ 60%
- 单作品平均曝光 ≥ 200 PV/月
- 群友提交作品后 7 日内至少获得 3 个点赞

## 2. 目标用户与优先级

| 优先级 | 用户 | 核心诉求 | 设计倾斜 |
|---|---|---|---|
| P0 | 群友（创作者+观众） | 看彼此在做什么、互相点赞收藏、秀作品 | 登录顺滑、提交低门槛、动态感强 |
| P1 | 外部潜在用户 | 找好玩的小程序/小游戏玩 | SEO 友好、分享友好、扫码即玩 |
| P2 | 潜在新群友 | 了解群气质，评估是否加入 | 首页"关于我们+加群"入口、作品质量即招牌 |

## 3. 功能范围

### 3.1 MVP（V1.0）必做

- 作品浏览：首页、分类页、标签页、详情页
- 点赞、收藏
- **用户名 + 密码注册登录**
- **邀请码注册机制**（控制初期只有群友能注册）
- 作品自助提交 + 后台审核
- 作者主页（头像、昵称、简介、作品列表）
- 标签与分类筛选
- 群主管理后台（审核、精选位、标签、用户、邀请码）

### 3.2 V1.1（二期）

- 邮箱验证码登录 + 忘记密码流程
- 精选位运营、排行榜（7 日/30 日热门）
- 作品数据看板（给作者看自己的 PV / 点赞趋势）
- 分享卡片生成（海报图，便于转发到朋友圈/小红书）

### 3.3 V1.2+（远期）

- GitHub OAuth 登录
- 评论（视生态健康度再决定要不要）

### 3.4 明确不做

- 微信登录 / 微信生态接入
- 内置试玩（小程序做不到，web 也带来安全风险）
- 关注、私信
- 付费 / 打赏
- Docker 化部署

## 5. 核心功能细节

### 5.1 作品展示

**作品卡片（列表用）**：封面图（4:3）、标题、作者头像+昵称、主标签 1 个、点赞数。

**作品详情字段**：

| 字段 | 必填 | 说明 |
|---|---|---|
| 名称 | ✓ | ≤30 字 |
| 类型 | ✓ | 单选：游戏 / 工具 / 社交 / AI 应用 / 其他 |
| 标签 | ✓ | 多选，最多 3 个 |
| 一句话简介 | ✓ | ≤30 字，用于卡片和分享 |
| 详细简介 | ✓ | ≤300 字 |
| 封面图 | ✓ | 1 张，建议 1200×900 |
| 截图 | ✓ | 3-6 张 |
| 微信小程序码 | 二选一 | 图片，详情页放大显示 |
| web 链接 | 二选一 | 外部跳转 |
| 补充说明 | - | 给审核员看，例如"这是测试账号 xxx" |

### 5.2 标签体系

**预设标签池**（群主维护）：休闲、益智、AI、社交、工具、效率、创意、单机、多人、沙雕、实验性……

- 群友提交时从预设池选，避免标签爆炸。
- 可申请新标签：进入后台队列，群主审核通过后加入预设池。
- 单个作品最多 3 个标签。

### 5.3 登录与权限

**V1.0 登录方式**：用户名 + 密码

**注册流程**：
1. 访问注册页，必填邀请码
2. 填写用户名（唯一，3-20 字符，字母数字下划线中文）
3. 设置密码（≥8 位）
4. 邮箱选填（用于后续找回密码，V1.0 不做邮箱验证）
5. 注册成功，邀请码核销

**密码安全**：
- bcrypt 加密存储（cost=12）
- 登录失败连续 5 次锁定账号 30 分钟
- 登录态：httpOnly cookie + JWT，有效期 30 天

**忘记密码 V1.0**：
- 有邮箱的 → 显示"请联系群主重置"，群主后台可手动重置
- V1.1 邮箱验证码上线后切换为自动流程

**角色**：
- 游客：只能浏览、不能点赞收藏
- 注册用户：浏览 + 点赞 + 收藏 + 提交作品
- 群主（admin）：全部 + 审核 + 运营 + 用户管理

### 5.4 邀请码机制

**作用**：控制 V1.0 只有群友能注册，防止垃圾提交。

**后台功能**：
- 批量生成（一次 10/50/100 个）
- 可设置类型：一次性 / 多次使用（设上限）
- 可设过期时间（默认 30 天）
- 查看使用情况（谁用了、什么时候用的）
- 作废（防止邀请码泄露后仍能注册）

**前台使用**：
- 注册页第一步填邀请码
- 核销后邀请码和使用者绑定，可追溯

### 5.5 提交与审核

**提交流程**：
1. 登录后访问"提交新作品"
2. 填写表单（字段见 5.1）
3. 提交后状态为"待审核"，创作者在用户中心可见

**审核流程**：
- 群主后台有"审核队列"，卡片形式一屏看完关键字段
- 三个操作：通过 / 驳回 / 编辑后通过
- 驳回必须填理由，站内通知 + 邮件（有邮箱）告知作者
- 通过后默认进"最新"流，不自动进精选
- 审核目标：单作品 30 秒内完成审核

**状态机**：
```
草稿（暂不实现，提交即进审核）
  ↓
待审核 ─┬─→ 已上架（live）
        ├─→ 已驳回（rejected，可修改后重提）
        └─→ 群主下架（unlisted，作者不可见）
```

### 5.6 点赞与收藏

- **点赞**：公开计数，详情页和卡片都显示。每人每作品只能点一次。
- **收藏**：私密，进用户中心"我的收藏"。
- 两者都必须登录。游客点击时弹登录引导。
- 反刷：同 IP + 同设备频率限制（每分钟 ≤ 30 次操作）。

### 5.7 作者主页

- URL：`/u/{slug}`，slug 由用户名自动生成（中文→拼音）或用户自定义
- 展示字段：头像、昵称、≤100 字简介、其他作品网格
- **不展示**联系方式，不做关注功能，避免骚扰和社交负担
- 作者可在账号设置里编辑昵称、简介、头像、slug

## 7. 技术方案

### 7.1 技术栈

- **前端**：Next.js 14+（App Router，SSR 用于 SEO）+ TypeScript + TailwindCSS
- **UI**：shadcn/ui（改造成圆角暖色调）+ 自定义插画资源
- **后端**：Next.js API Routes（单体，够用；不想单体就拆 NestJS）
- **数据库**：PostgreSQL 16
- **缓存**：Redis 7（点赞计数、session、限流）
- **ORM**：Prisma 或 Drizzle（Drizzle 更轻）

### 7.2 部署架构

**阿里云轻量应用服务器（已备案）**，全部组件同机部署，**不使用 Docker**：

```
阿里云轻量服务器
├── Nginx（443/80，反向代理 + HTTPS + 静态资源）
│   ├── 证书：Let's Encrypt（certbot 自动续期）
│   └── /uploads/ 路径直接指向本地图片目录
├── Node.js（pm2 守护，监听 3000）
│   └── Next.js 应用（前端 SSR + API）
├── PostgreSQL 16（监听 127.0.0.1:5432）
├── Redis 7（监听 127.0.0.1:6379）
└── 图片本地存储：/var/aircade/uploads/
```

**规格建议**：2 核 4G / 4M 带宽 / 60G SSD 起步。日活 500 以内压力不大。

**进程管理**：pm2 管理 Node 进程（自动重启、开机启动、日志滚动）。

### 7.3 图片存储

- **V1.0 存服务器本地磁盘**：`/var/aircade/uploads/{year}/{month}/{hash}.{ext}`
- 上传时做压缩：封面 ≤ 500KB，截图 ≤ 800KB，统一转 webp
- Nginx 直接服务 `/uploads/` 路径，开启 1 年缓存头
- 预留迁移口子：图片 URL 存数据库时用相对路径，后期切 OSS 只需改 base URL

### 7.4 备份方案

**备份到 GitHub Private Repo**（`aircade-backup`）：

- 数据库：每日 3:00 AM 执行 `pg_dump | gzip`，按日期命名 `db-YYYY-MM-DD.sql.gz`
- 图片：每周日打包 `/var/aircade/uploads/` 为 `uploads-YYYY-WW.tar.gz`
- 用 cron + shell 脚本完成，git commit & push
- 利用 git 历史做版本管理，无需手动清理
- **限制提醒**：GitHub 单文件 100MB、仓库建议 ≤1GB。图片量大到接近上限时改用 OSS。

**备份脚本示例**（`/opt/aircade/backup.sh`）：

```bash
#!/bin/bash
set -e
BACKUP_DIR=/opt/aircade-backup
cd $BACKUP_DIR
DATE=$(date +%Y-%m-%d)
pg_dump aircade | gzip > db-$DATE.sql.gz
git add . && git commit -m "backup $DATE" && git push
```

加到 crontab：`0 3 * * * /opt/aircade/backup.sh >> /var/log/aircade-backup.log 2>&1`

### 7.5 部署流程（手工，单人开发够用）

1. 本地 `pnpm build` 通过
2. `git push` 到 main 分支
3. SSH 到服务器：`git pull && pnpm install && pnpm build && pm2 restart aircade`
4. 后续可加 GitHub Actions 自动部署（V1.1 再搞）

### 7.6 安全基线

- HTTPS 强制（Nginx 301 重定向）
- 数据库和 Redis 只监听 127.0.0.1
- 定期系统更新（`unattended-upgrades`）
- 防火墙只开 22/80/443
- SSH 禁用密码登录，只用密钥
- 所有用户输入做校验和转义
- 文件上传白名单类型（jpg/png/webp），扫描 magic number 不信 MIME
- 管理后台 `/admin` 路由加 IP 白名单（可选，看你要不要从家和公司都能登）

## 8. 视觉与交互原则

**调性**：可爱派对、圆角、暖色、插画感。参考：即刻 App、Bento 卡片、Notion Calendar 插画风。

- **主色**：暖橙 `#FF9F6B` / 奶黄 `#FFE9B8`
- **辅色**：薄荷绿 `#9FE3C9` / 樱粉 `#FFB7C5`
- **中性**：奶白 `#FFFBF5` / 深棕 `#3D2E1F`（比纯黑柔和）
- **字体**：中文霞鹜文楷 / HarmonyOS Sans，英文 Inter
- **圆角**：按钮 12px、卡片 20px、输入框 10px
- **空态**：用插画，不用纯文字，保持调性
- **微动效**：点赞时爱心弹跳、卡片 hover 微浮起、页面切换淡入
- **移动端优先**：主受众在手机上，H5 体验必须顺滑

## 9. 非功能需求

- **性能**：首屏 LCP < 2s（国内），图片 CDN + webp
- **SEO**：作品详情页 SSR，title/description/og 三件套完整
- **可访问性**：基本 a11y（alt 文字、焦点环、键盘导航）
- **合规**：提交前勾选《内容规范》、提供举报入口、用户协议 + 隐私政策
- **可维护性**：单人运营，审核流程单作品 ≤ 30 秒完成
- **监控**：pm2 自带进程监控；错误日志每日邮件摘要（V1.1）

## 12. 风险与应对

| 风险 | 可能性 | 影响 | 应对 |
|---|---|---|---|
| 冷启动作品不够 | 中 | 首页空荡 | 上线前自己+核心群友攒够 15+ |
| 邀请码泄露导致垃圾注册 | 中 | 审核压力 | 邀请码一次性为主、可随时作废 |
| 图片占满磁盘 | 低（初期） | 服务中断 | 监控磁盘使用率，阈值 70% 时切 OSS |
| 服务器挂了 | 低 | 全站不可用 | 轻量服务器自带快照，每周一次 |
| 群主一人审核跟不上 | 中（后期） | 积压严重 | V1.1 可设置"群友作品自动过，新人作品需审" |
| 备份仓库超限 | 低 | 备份失败 | 监控备份大小，超 500MB 迁 OSS |
